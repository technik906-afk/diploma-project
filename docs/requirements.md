# Требования к инфраструктуре MediaWiki

## Шаблон сбора требований

| Параметр | Значение |
|----------|----------|
| 🐧 **ОС сервисов проекта** | Ubuntu 22.04 LTS |
| 📦 **Необходимые библиотеки и фреймворки** | PHP 8.x, PostgreSQL 14+, Nginx |
| 👥 **Планируемая нагрузка** | ~40 человек (центральный офис) |
| 📋 **Специфика приложения** | Вики-система для внутренней документации |
| 🌐 **Интерфейс взаимодействия** | HTTP-сервис (веб-интерфейс) |
| 🔧 **Вспомогательные сервисы** | PostgreSQL, Nginx, Zabbix, Backup |

---

## Детальные требования

### 1. 🐧 ОС и базовое ПО

| Компонент | Технология | Версия |
|:---|:---|:---|
| Операционная система | Ubuntu | 22.04 LTS |
| Веб-сервер | Nginx | последняя стабильная |
| Язык приложения | PHP | 8.x |
| База данных | PostgreSQL | 14+ |

### 2. 👥 Нагрузка

| Параметр | Значение |
|:---|:---|
| Количество пользователей | ~40 человек (центральный офис) |
| Перспектива | Масштабирование на всю компанию (офис + удалённые) |
| Ожидаемая нагрузка | Низкая/средняя (демонстрационный проект) |

### 3. 🏗️ Архитектура

#### Общая схема

```
┌─────────────┐
│ Пользователи │
└──────┬──────┘
       │ HTTP
       ▼
┌──────────────┐
│  Nginx LB    │  (LB-01, Public IP)
│  111.88.241.156│
└──┬───────┬───┘
   │ HTTP  │ HTTP
   ▼       ▼
┌──────┐ ┌──────┐
│APP-01│ │APP-02│  (MediaWiki, PHP-FPM)
│10.0.2.10│ │10.0.2.11│
└──┬───┘ └──┬───┘
   │        │
   │  TCP:5432
   ▼        ▼
┌──────────────┐  ┌──────────────┐
│   DB-01      │  │   DB-02      │
│  10.0.3.10   │──▶│  10.0.3.11  │
│  (Master)    │  │  (Replica)   │
└──────────────┘  └──────────────┘
```

#### Вспомогательные сервисы

| Сервис | Адрес | Назначение |
|--------|-------|------------|
| **Zabbix** | 10.0.4.10 | Мониторинг всех сервер |
| **Backup** | 10.0.5.10 | Ежедневные бэкапы ФС и БД |

#### Сетевая топология

| Сервер | IP | Роль |
|--------|-----|------|
| LB-01 | 111.88.241.156 (public) | Балансировщик, jump-host |
| APP-01 | 10.0.2.10 | MediaWiki + PHP-FPM |
| APP-02 | 10.0.2.11 | MediaWiki + PHP-FPM |
| DB-01 | 10.0.3.10 | PostgreSQL Master |
| DB-02 | 10.0.3.11 | PostgreSQL Replica |
| Zabbix | 10.0.4.10 | Zabbix Server |
| Backup | 10.0.5.10 | Сервер бэкапов |

### 4. 🛡️ Высокая доступность

| Компонент | Решение | Описание |
|:---|:---|:---|
| **Балансировщик** | Nginx upstream | Распределение трафика между APP-01 и APP-02 |
| **Приложение** | 2 экземпляра | Отказоустойчивость: при падении одного — работает второй |
| **База данных** | Streaming replication | Master → Replica, возможность переключения |
| **Бэкапы** | Ежедневные | Бэкапы ФС и БД с хранением 7 дней |

### 5. 📊 Мониторинг

| Параметр | Значение |
|:---|:---|
| **Система** | Zabbix Server 6.x/7.x |
| **Метрики** | Доступность HTTP, CPU/RAM, Disk, статус репликации |
| **Оповещения** | При отклонении от нормы |

### 6. 💾 Резервное копирование

| Параметр | Значение |
|:---|:---|
| **Бэкап ФС** | Ежедневно, архив `/var/www/mediawiki` (tar.gz) |
| **Бэкап БД** | Ежедневно, `pg_dump` (sql.gz) |
| **Хранение** | 7 дней |
| **Место хранения** | Отдельный сервер BACKUP-01 |
| **Расписание** | БД — 2:00, ФС — 3:00 (cron) |

### 7. 🔒 Безопасность

| Мера | Описание |
|:---|:---|
| **SSH доступ** | Только по ключам, через jump-host (LB-01) |
| **Public IP** | Только у LB-01, все остальные — внутренние |
| **Security Groups** | Ограничение трафика между компонентами |
| **БД** | Доступ только от APP-серверов, BACKUP и ZABBIX |

### 8. 💰 Экономия ресурсов

| Мера | Экономия | Описание |
|:---|:---|:---|
| **Прерываемые ВМ** | ~60-70% | Все 7 серверов |
| **HDD диски** | ~50-60% | Вместо SSD |
| **Итого** | **~70%** | ~924 руб/мес |

---

## Обоснование архитектурных решений

| Решение | Обоснование |
|---------|-------------|
| 2 сервера приложений | Отказоустойчивость, балансировка нагрузки |
| Репликация PostgreSQL | Защита от потери данных, возможность переключения на реплику |
| Отдельный backup-сервер | Изоляция бэкапов, долгосрочное хранение |
| Zabbix мониторинг | Раннее обнаружение проблем |
| Прерываемые ВМ | Экономия ~70% для демонстрационного проекта |
| HDD диски | Экономия, достаточно для низкой нагрузки |
| Только LB-01 с Public IP | Безопасность (единственная точка входа), экономия на IP |

## Критерии приёмки

| № | Критерий | Статус |
|---|----------|--------|
| 1 | MediaWiki доступна по HTTP через балансировщик | ✅ |
| 2 | При выключении одного APP-сервера сервис продолжает работать | ⏳ |
| 3 | Репликация PostgreSQL работает корректно | ✅ |
| 4 | Бэкапы создаются по расписанию | ✅ |
| 5 | Zabbix мониторит все серверы | ✅ |
| 6 | SSH доступ работает через jump-host | ✅ |
