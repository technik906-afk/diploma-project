# Требования к инфраструктуре MediaWiki

## Шаблон сбора требований

| Параметр | Значение |
|----------|----------|
| **ОС сервисов проекта** | Ubuntu 22.04 LTS |
| **Необходимые библиотеки и фреймворки** | PHP 8.x, PostgreSQL 14+, Nginx |
| **Планируемая нагрузка (количество клиентов)** | ~40 человек (центральный офис) |
| **Специфика приложения** | Вики-система для внутренней документации |
| **Интерфейс взаимодействия с пользователем** | HTTP-сервис (веб-интерфейс) |
| **Вспомогательные сервисы** | PostgreSQL, Nginx, Zabbix, Backup |

---

## Детальные требования

### 1. ОС и базовое ПО

- **Операционная система:** Ubuntu 22.04 LTS
- **Веб-сервер:** Nginx (балансировка + приложение)
- **Язык приложения:** PHP 8.x
- **База данных:** PostgreSQL 14+

### 2. Нагрузка

- **Количество пользователей:** ~40 человек (центральный офис)
- **Перспектива:** масштабирование на всю компанию (офис + удалённые сотрудники)
- **Ожидаемая нагрузка:** низкая/средняя (демонстрационный проект)

### 3. Архитектура

```
Пользователи → Nginx LB → MediaWiki App (x2) → PostgreSQL (Master + Replica)
                                      ↓
                               Zabbix (мониторинг)
                                      ↓
                               Backup Server (бэкапы)
```

### 4. Высокая доступность

- **Балансировщик:** Nginx (распределение трафика между APP-01 и APP-02)
- **Приложение:** 2 экземпляра MediaWiki для отказоустойчивости
- **База данных:** PostgreSQL репликация (Master → Replica)
- **Бэкапы:** Ежедневные бэкапы ФС и БД

### 5. Мониторинг

- **Система:** Zabbix
- **Метрики:**
  - Доступность HTTP-сервиса (код ответа, время ответа)
  - Загрузка CPU/RAM серверов
  - Свободное место на дисках
  - Статус PostgreSQL репликации

### 6. Резервное копирование

- **Бэкап ФС:** Ежедневно, архив /var/www/mediawiki
- **Бэкап БД:** Ежедневно, pg_dump
- **Хранение:** 7 дней
- **Место хранения:** Отдельный сервер (BACKUP-01)

### 7. Безопасность

- **SSH доступ:** Только по ключам, через jump-host
- **Security Groups:** Ограничение трафика между компонентами
- **БД:** Доступ только от APP-серверов и BACKUP

### 8. Экономия ресурсов

- **Тип ВМ:** Прерываемые (preemptible) — дешевле до 70%
- **Тип диска:** Сетевой HDD — дешевле SSD
- **Общая стоимость:** ~924 руб/мес (ориентировочно)

---

## Обоснование архитектурных решений

| Решение | Обоснование |
|---------|-------------|
| 2 сервера приложений | Отказоустойчивость, балансировка нагрузки |
| Репликация PostgreSQL | Защита от потери данных, возможность переключения |
| Отдельный backup-сервер | Изоляция бэкапов, долгосрочное хранение |
| Zabbix мониторинг | Раннее обнаружение проблем |
| Прерываемые ВМ | Экономия для демонстрационного проекта |
| HDD диски | Экономия, достаточно для низкой нагрузки |

---

## Критерии приёмки

- [ ] MediaWiki доступна по HTTP через балансировщик
- [ ] При выключении одного APP-сервера сервис продолжает работать
- [ ] Репликация PostgreSQL работает корректно
- [ ] Бэкапы создаются по расписанию
- [ ] Zabbix мониторит все серверы
- [ ] SSH доступ работает через jump-host
