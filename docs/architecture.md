# Архитектура инфраструктуры MediaWiki

## Обзор

Инфраструктура развёрнута в Yandex Cloud и состоит из 7 виртуальных машин, объединённых в единую сеть VPC.

---

## Схема архитектуры

```
                                    INTERNET
                                        │
                                        ▼
                            ┌─────────────────────┐

                                       │
                    ┌──────────────────┴──────────────────┐
                    ▼                                     ▼
        ┌───────────────────┐               ┌───────────────────┐

                    ┌───────────────┴───────────────┐

        ┌───────────────────┐           ┌───────────────────┐
```

## Компоненты

### 1. Балансировщик нагрузки (LB-01)

| Параметр | Значение |
|----------|----------|
| IP | 10.0.1.10 (Private), Public IP |
| vCPU/RAM | 2 / 2 GB |
| ПО | Nginx (upstream балансировка) |
| Порты | 80 (HTTP), 443 (HTTPS), 22 (SSH) |

**Функции:**
- Распределение трафика между APP-01 и APP-02
- SSH jump-host для доступа к внутренней сети
- Единственная точка входа из интернета

---

### 2. Серверы приложений (APP-01, APP-02)

| Параметр | Значение |
|----------|----------|
| IP | 10.0.2.10, 10.0.2.11 |
| vCPU/RAM | 2 / 2 GB |
| ПО | MediaWiki, PHP 8.x, Nginx |
| Порты | 80 (HTTP от LB), 22 (SSH) |

**Функции:**
- Хостинг MediaWiki
- Обработка веб-запросов
- Подключение к PostgreSQL

---

### 3. Серверы базы данных (DB-01, DB-02)

| Параметр | Значение |
|----------|----------|
| IP | 10.0.3.10 (Master), 10.0.3.11 (Replica) |
| vCPU/RAM | 2 / 2 GB |
| ПО | PostgreSQL 14+ |
| Порты | 5432 (PostgreSQL), 22 (SSH) |

**Функции:**
- Хранение данных MediaWiki
- Репликация Master → Replica
- Резервное копирование

---

### 4. Сервер мониторинга (ZABBIX-01)

| Параметр | Значение |
|----------|----------|
| IP | 10.0.4.10 |
| vCPU/RAM | 2 / 2 GB |
| ПО | Zabbix Server, Zabbix Agent |
| Порты | 80 (Web), 10050/10051 (Zabbix), 22 (SSH) |

**Функции:**
- Мониторинг доступности сервисов
- Сбор метрик производительности
- Оповещения о проблемах

---

### 5. Сервер резервного копирования (BACKUP-01)

| Параметр | Значение |
|----------|----------|
| IP | 10.0.5.10 |
| vCPU/RAM | 2 / 2 GB |
| ПО | rsync, SSH |
| Порты | 22 (SSH от APP/DB) |

**Функции:**
- Хранение бэкапов ФС и БД
- Долгосрочное хранение (7 дней)

## Сетевая архитектура

### VPC и подсети

```
VPC: mediawiki-vpc (10.0.0.0/16)

┌────────────────────────────────────────────────────┐
│ Подсеть       │ CIDR        │ Назначение           │
├────────────────────────────────────────────────────┤
│ lb-subnet     │ 10.0.1.0/24 │ Балансировщик        │
│ app-subnet    │ 10.0.2.0/24 │ Приложения           │
│ db-subnet     │ 10.0.3.0/24 │ База данных          │
│ zabbix-subnet │ 10.0.4.0/24 │ Мониторинг           │
│ backup-subnet │ 10.0.5.0/24 │ Бэкапы               │
└────────────────────────────────────────────────────┘
```

### Security Groups

| Группа | Описание | Правила |
|--------|----------|---------|
| sg-lb | Балансировщик | HTTP/HTTPS из интернета, SSH от админа |
| sg-app | Приложения | HTTP от LB, SSH от админа |
| sg-db | БД | PostgreSQL от APP/BACKUP, SSH от админа |
| sg-zabbix | Мониторинг | HTTP из интернета, Zabbix от внутренних, SSH от админа |
| sg-backup | Бэкапы | SSH от APP/DB |

---

## Потоки трафика

```
1. Пользователь → LB-01:80 → APP-01/APP-02:80 → DB-01:5432
2. Админ → LB-01:22 → (jump) → Все серверы:22
3. Все серверы → ZABBIX-01:10050/10051 (мониторинг)
4. APP-01/DB-01 → BACKUP-01:22 (бэкапы)
```

## Инфраструктура как код (IaC)

### Terraform

- **providers.tf** — провайдер Yandex Cloud
- **variables.tf** — переменные конфигурации
- **main.tf** — основные ресурсы
- **vpc.tf** — сеть и подсети
- **security-groups.tf** — группы безопасности
- **compute.tf** — виртуальные машины
- **outputs.tf** — выходные данные

### Ansible

- **site.yml** — главный playbook
- **inventory/hosts.ini** — инвентори
- **playbooks/** — playbooks для каждого компонента
- **roles/** — роли для компонентов

## Масштабирование

### Горизонтальное (приложение)

- Добавление APP-серверов
- Обновление конфигурации Nginx upstream

### Вертикальное (ресурсы)

- Увеличение vCPU/RAM через Terraform
- Увеличение дискового пространства

### База данных

- Добавление реплик для чтения
- Шардинг при необходимости
